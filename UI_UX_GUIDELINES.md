# AFOS Studio UI / UX Guidelines

## Purpose

This document defines the complete user experience, navigation system, design language, and interaction rules for AFOS Studio.

AFOS must feel like professional filmmaking software while remaining simple enough for beginners.

---

# Design Philosophy

AFOS is not a chatbot.

AFOS is not a dashboard full of statistics.

AFOS is a filmmaking workspace.

Every screen should answer one question:

"What should I do next?"

---

# Core Design Principles

1. One primary action per screen.
2. Never overwhelm the user.
3. Guide the user through production.
4. Show production progress at all times.
5. Hide advanced settings until requested.
6. Mobile-first design.
7. Professional cinematic appearance.
8. Fast navigation.

# Notifications

Use short messages.

Examples:

Storyboard Ready

Character Approved

Export Finished

# Expert Mode

AFOS has two modes.

Director Mode

Simple interface.

No prompts.

No AI settings.

Expert Mode

Shows:

Prompts

Generation Parameters

Model Selection

Advanced Editing

Default mode is Director Mode.
